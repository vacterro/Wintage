# ADR-001: Animation taming is safe; the v1.0.9 rc-motion fear was false

## Status
Accepted (v1.2.0).

## Context
The theme forces motion to near-zero for the vintage no-motion aesthetic.
v1.0.7 forced `animation-duration` alongside transitions. v1.0.9 reverted the
animation forcing, blaming it for the chat.qwen.ai dropdown becoming unclickable.
That blame was a hypothesis, never verified. v1.1.1 later proved the ACTUAL
cause was the forced `transition-property` list (top/left/width), which broke
Ant Design rc-trigger's popup positioning. Animations were exonerated but never
re-enabled, leaving the theme's "zero animations" claim false.

## Decision
Force `animation-duration: 0.001s` and `animation-delay: 0s` on
`*, *::before, *::after` in both GLOBAL_CSS and SHADOW_CSS.

- `0.001s`, not `0s`: a genuine instant animation lifecycle reliably fires
  `animationstart`/`animationend`, so `animationend`-driven state machines
  (rc-motion) still advance. `0s` has engine edge cases where events may not fire.
- Applies at `document-start`, so entrance animations are instant from first
  paint (truly invisible, not a cut-off flash).

## Verification (live, chat.qwen.ai, Chromium)
- rc-motion "+" dropdown under the rule: opens, menu item hit-test
  `elementFromPoint` resolves to the item (`itemClickable=true`), 7 items,
  page responsive. Not broken.
- Scan of 1757 real page elements: 0 had `opacity:0` + text + active animation
  (i.e. 0 content hidden by snapback).

## Consequences / bounded risks
- Snapback: a base `opacity:0` element revealed only by a non-fill-mode
  animation reverts to invisible after the (now-instant) animation ends. This
  only affects already-broken authoring (correct reveals use
  `fill-mode:forwards` or a final base state). 0 occurrences on real qwen.
- Infinite spinners: Chromium coalesces `animationiteration` to <=1 per frame,
  so no event flood. Spinners freeze/snap rather than spin — acceptable under
  no-motion aesthetic.

## Rule for future agents
Do NOT re-revert animation forcing on a hunch. If a dropdown/popup breaks,
suspect the `transition-property` list (positioning) FIRST — that has broken
real sites twice; animation forcing has broken zero, verified.
